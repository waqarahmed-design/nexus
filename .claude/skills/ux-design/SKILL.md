# UX Design

Comprehensive UX design guidance across four domains: interaction design, wireframing & prototyping, usability & accessibility, and responsive & adaptive design. Covers 29 methods for designing experiences that are intuitive, inclusive, and work across contexts.

## Method Selection Guide

| Question | Method |
|----------|--------|
| How does a user move through the product? | User Flow Creation |
| How does a user complete a specific task? | Task Flow Design |
| What does every state of this element look like? | State Design |
| What happens when things go wrong or are empty? | Edge Case Handling |
| How do we avoid overwhelming users with complexity? | Progressive Disclosure |
| How do users know this is clickable/tappable? | Affordance Design |
| How does the system communicate what just happened? | Feedback Design |
| What does the UI text say? | Microcopy |
| How do we quickly explore layout options? | Low-Fidelity Wireframing |
| How do we document the final layout precisely? | High-Fidelity Wireframing |
| How do we test navigation and transitions? | Interactive Prototyping |
| How do we validate a concept in hours? | Rapid Prototyping |
| How do we test concepts before any screen exists? | Paper Prototyping |
| How do we build a realistic prototype with logic? | Code-Based Prototyping |
| Does this meet accessibility standards? | WCAG Compliance |
| Can users navigate without a mouse? | Keyboard Navigation |
| Does this work with VoiceOver / TalkBack? | Screen Reader Optimization |
| Do colors pass contrast ratios? | Color Contrast |
| Is focus management logical and visible? | Focus Management |
| Does this work for diverse abilities and contexts? | Inclusive Design |
| Are we asking too much of users' mental bandwidth? | Cognitive Load Management |
| Does the design prevent mistakes? | Error Prevention |
| Can users easily recover from errors? | Forgiveness Design |
| Does this work on the smallest screen first? | Mobile-First Design |
| How does layout adapt across screen sizes? | Breakpoint Strategy |
| Are touch targets large enough to tap accurately? | Touch Target Sizing |
| How do grids and images adapt fluidly? | Responsive Layout Patterns |
| Are we following iOS / Android / Web conventions? | Platform-Specific Patterns |
| Does experience feel continuous across devices? | Cross-Device Continuity |

## Domain Sequencing

```
Interaction Design (define the experience logic)
  └── User Flow → Task Flow → State Design → Edge Cases
      → Progressive Disclosure → Affordance → Feedback → Microcopy

Wireframing & Prototyping (make it tangible)
  └── Low-Fi Wireframes → High-Fi Wireframes
      → Interactive Prototype → Rapid / Paper / Code Prototype

Usability & Accessibility (make it inclusive)
  └── WCAG → Color Contrast → Keyboard Nav → Screen Reader
      → Focus Management → Inclusive Design
      → Cognitive Load → Error Prevention → Forgiveness

Responsive & Adaptive (make it universal)
  └── Mobile-First → Breakpoint Strategy → Touch Targets
      → Responsive Patterns → Platform Conventions → Cross-Device
```
