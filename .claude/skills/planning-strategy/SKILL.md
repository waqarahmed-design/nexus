# Planning & Strategy

Design planning and strategy methods across three domains: product strategy, information architecture, and project planning. Covers 17 methods for defining direction, structuring information, and running design work.

## Method Selection Guide

| Question | Method |
|----------|--------|
| What problem are we solving? | Problem Definition |
| Which opportunity to pursue first? | Opportunity Assessment |
| Which features to build next? | Feature Prioritization (RICE / MoSCoW / Kano) |
| What's the design work over next 6 months? | Roadmap Planning |
| How will we know if we succeeded? | Success Metrics Definition |
| Why would users choose this product? | Value Proposition Design |
| How should content be organized? | Content Strategy |
| What's the overall product structure? | Site Mapping |
| How should categories be named/grouped? | Taxonomy Design |
| How do users move through the product? | Navigation Design |
| How do users find things? | Search Design |
| How do users mentally model the domain? | Mental Model Mapping |
| How long will this design work take? | Design Scoping |
| Who needs to be involved? | Resource Planning |
| What could go wrong? | Risk Assessment |
| How to solve a problem in a focused week? | Design Sprint Facilitation |
| How to run a collaborative session? | Workshop Facilitation |

## Domain Sequencing

```
Strategy (define what to build)
  └── Problem Definition → Opportunity Assessment → Feature Prioritization
      → Success Metrics Definition → Value Proposition Design → Roadmap Planning

Information Architecture (define how to structure it)
  └── Mental Model Mapping → Content Strategy → Site Mapping
      → Taxonomy Design → Navigation Design → Search Design

Project Planning (define how to execute)
  └── Design Scoping → Resource Planning → Risk Assessment
      → Workshop Facilitation / Design Sprint Facilitation
```
